# 🎧 DJ Video Maker

DJ用の Remix / Edit の音源を入れると、**公式MVを自動で探して曲に同期したMP4**を作るツールです。
Serato / VirtualDJ 用の**ビートグリッドとCue**も書き込み済みで出力します。

created by DJ SOPY / [@sousouagain](https://twitter.com/sousouagain)

---

## できること

- 曲名・アーティスト名から**YouTubeの公式MVを自動検索**してダウンロード
- Remix（テンポ・アレンジ違い）でも、**ボーカルを分離して口の動きに合わせて配置**（リップシンク）
- 歌っていない区間（ドロップ／VJパート）は別映像で自然に埋める
- Extended化（イントロ追加）・タイトルテロップの付与
- BPM・ビートグリッド・Cueを書き込んで出力（Serato / VirtualDJ でそのまま使える）

## 動作環境

- macOS（Apple Silicon / Intel）
- インターネット接続（初回セットアップとMV取得に必要）
- **初回のみ管理者アカウント**が必要（Homebrewの導入に使うため。macOSの仕様）

必要なツール（ffmpeg / yt-dlp / Python ライブラリ）は**初回起動時に自動でインストール**されます。
初回は数分〜30分ほどかかります（AIモデルのダウンロードを含む）。2回目以降は速いです。

## インストール

### 方法A：git clone（おすすめ）

```bash
git clone https://github.com/<ユーザー名>/<リポジトリ名>.git
cd <リポジトリ名>
```

clone した場合は**Gatekeeperの隔離マークが付かない**ので、そのまま `DJ_Video_Maker.command` をダブルクリックできます。

### 方法B：ZIPでダウンロード

「Code」→「Download ZIP」で落として展開します。
この場合、macOSがネット由来のファイルをブロックするため、**初回だけ**次のどちらかが必要です。

- `DJ_Video_Maker.command` を**右クリック →「開く」**→ 警告が出るのでもう一度「開く」
- または、ターミナルで隔離マークを外す：

  ```bash
  xattr -cr "<展開したフォルダのパス>"
  ```

一度起動すれば、次回からはダブルクリックだけで開けます（起動時に自動で隔離を解除します）。

## 使い方

1. フォルダ内のファイルを**全部同じ場所に置いたまま**、`DJ_Video_Maker.command` をダブルクリック
2. ターミナルが開くので、案内に従って**音楽ファイルをドラッグ&ドロップ → Enter**（複数可）
3. 出力先・Extended化・テロップの有無を選んだら、あとは待つだけ

完成したMP4は既定で**デスクトップ**に出ます。

### ファイル名のコツ

`アーティスト名 - 曲名` の形式にしておくと、MVの自動検索精度が上がります。
（音源のタグ情報も読むので、タグが正しければファイル名は多少雑でも大丈夫です）

## 困ったときは

| 症状 | 原因と対処 |
|---|---|
| 「開発元を検証できないため開けません」 | ネット由来ファイルのブロック。**右クリック →「開く」**を2回 |
| 「壊れているため開けません」 | 隔離マーク。`xattr -cr <フォルダ>` を実行してから右クリック →「開く」 |
| ダブルクリックでテキストエディタが開く／何も起きない | 実行権限が外れている。`chmod +x DJ_Video_Maker.command` |
| 「Need sudo access」「Homebrewのインストールに失敗」 | そのアカウントが**管理者ではない**。システム設定 → ユーザとグループ → 「このユーザを管理者として設定」をON → 再起動（データは消えません） |
| YouTubeが403／ボット判定される | ログイン済みブラウザのCookieを使う設定を見直す。`~/.dj_video_maker/config.json` を削除すると、次回起動時にブラウザを選び直せます |
| 動画がダウンロードできない | yt-dlp が古い可能性。`brew upgrade yt-dlp` |
| 口パクが合わない／原曲MVがそのまま流れる | ボーカル分離（Demucs）が動いていない可能性。ターミナルのログに `分離方式: remix=demucs / MV=demucs` と出ているか確認してください |
| 何をやってもおかしい | `修復_初回からやり直し.command` をダブルクリック（環境を作り直します） |

Cookieが読めない場合は自動でCookie無しに切り替えて続行するので、それだけで停止することはありません。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `DJ_Video_Maker.command` | 起動用。初回セットアップもここが行う |
| `dj_maker_core.py` | 本体（MV検索・ダウンロード・配置・書き出し） |
| `lipsync_pro.py` | 高精度リップシンク（HuBERT + subsequence DTW + Viterbi経路） |
| `vocal_sync.py` | ボーカル分離（Demucs）ベースのリップシンク |
| `mouth_sync.py` | 口の動き解析（歌っていない区間の口パク映像を回避） |
| `修復_初回からやり直し.command` | Python環境を作り直す修復スクリプト |
| `tests/` | 回帰テスト（`python3 -m pytest tests/`） |

`.py` は4つとも `.command` と**同じフォルダ**に置いてください。欠けると機能が落ちます。

---

## 開発メモ

```bash
python3 -m pytest tests/ -q
```

- 音声・映像の処理は ffmpeg に統一（`torchaudio.save` は使わない＝torchcodec 非依存）
- Demucs は Python API（`apply_model`）で直接呼び出し、stem をテンソルのまま受け取る
- 映像のカットはフレーム数で厳密に指定（秒指定だと端数が積み上がって尺がずれる）
