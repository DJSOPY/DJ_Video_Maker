🎧 DJ Video Maker 説明書
YouTube自動選択版

DJ用のRemix/EditのMP3・動画を入れると、公式MVを自動で探して曲に同期したMP4を作るツール。Serato / VirtualDJ 用のビートグリッド・Cueも書き込み済みで出力します。

✅ 使い方（3ステップ）

  1. フォルダの中のファイルを全部同じ場所に置いたまま、DJ_Video_Maker.command をダブルクリック
  2. 黒い画面（ターミナル）が開くので、案内に従って曲ファイルをドラッグ&ドロップ → Enter。あとはMVを自動で探して同期します
  3. 待つだけ。完成したMP4はデスクトップに出ます
初回だけ、必要なツールの自動インストールで数分〜30分かかります（2回目以降は速い）。ネット接続が必要です。

🚨 「.command が開けない！」を直す 最重要

macOSは、ネットからもらった .command を最初の1回だけブロックします。故障ではありません。症状別にどれかで開けます。

A．「開発元を検証できないため開けません」と出る

  1. DJ_Video_Maker.command を右クリック（または control＋クリック）
  2. メニューの「開く」を選ぶ
  3. 警告が出るが、もう一度「開く」

→ 一度これで開けば、次からはダブルクリックでOK。

B．Aの「開く」が出てこない／それでもダメ

  1. アップルメニュー →「システム設定」→「プライバシーとセキュリティ」
  2. 下の方の「"DJ_Video_Maker.command"は開発元を確認できないため…」の横の「このまま開く」
  3. もう一度ダブルクリック

C．ダブルクリックするとテキストエディタで開いてしまう／何も起きない

＝実行の権限が外れています。ターミナル（Spotlightで「ターミナル」と検索）で：

chmod +x
※末尾に半角スペースが入った状態でコピーされます。貼ったらEnterせず、次へ
  1. 上の「コピー」を押してターミナルに貼る（まだEnterしない）
  2. .command ファイルをターミナルの画面にドラッグ&ドロップ（パスが自動で入る）
  3. Enter → もう一度ダブルクリック

D．「壊れているため開けません。ゴミ箱に入れる必要があります」と出る

ダウンロード時に付く隔離マークのせいです。同じ要領で：

xattr -cr
※末尾に半角スペース入り。貼ったらEnterせず、フォルダごとドラッグ&ドロップ → Enter

→ フォルダ内すべての隔離が外れます。そのあとAの手順で開いてください。

E．「適切なアクセス権限がないために実行できません」と出る

C と同じ直し方です（chmod +x → ファイルをドラッグ&ドロップ → Enter）。ZIPやAirDrop経由で権限が剥がれた時に出ます。

🔑 「Need sudo access」「Homebrewのインストールに失敗」と出る

Need sudo access on macOS (e.g. the user needs to be an Administrator)! ❌ Homebrewのインストールに失敗しました

これはネットの問題ではなく、使っているMacのアカウントが「管理者」ではないのが原因です。初回セットアップだけは管理者権限が必須です（macOSの仕様）。

安心してください：「管理者にする」はアカウントの切り替えではなく、今のアカウントに権限を足すだけ。音楽・写真・書類などのデータは一切消えません。心配なら、セットアップが終わった後に「通常」へ戻してもOK（入れたツールはそのまま使えます）。
  1. アカウントを管理者にする
    アップルメニュー → システム設定 → ユーザとグループ → 自分のアカウントの (i) → 「このユーザを管理者として設定」をON（Macの管理者パスワードが必要）→ 再起動
  2. Homebrewを手動で1回入れる（再起動後、ターミナルにこれを貼ってEnter） 
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    途中で Password: と出たらMacのパスワードを入力（画面には表示されないが打てています）→ Enter。「Press RETURN…」と出たら Enter。
  3. もう一度 .command をダブルクリック → 続きが自動で進みます

📡 「HTTP Error 403: Forbidden」「映像ファイルが見つかりません」と出る

ERROR: unable to download video data: HTTP Error 403: Forbidden ❌ 映像ファイルが見つかりません

YouTube側がダウンロードを一時的に拒否したエラーです。ツールやMacの故障ではありません（YouTubeは仕様を頻繁に変えるため、たまに起きます）。上から順に試してください：

  1. ChromeでYouTubeにログインする（一番効きます）
    Chromeを開く → YouTubeにログイン → 動画を1本普通に再生 → .command を再実行
    （このツールはChromeのログイン情報を使うため、未ログインだとボット扱いされやすい）
  2. ターミナルでこれを実行してから再実行： 
    brew upgrade yt-dlp
  3. それでもダメなら数時間〜1日待って再実行（対策版が自動配信されます。大抵数日以内に直ります）
  4. 別の公式動画のURLで試す（URL版の場合）

🏷 曲ファイルの名前の付け方（正しいMVを見つけるコツ）

ツールは ①ファイル内のタグ（曲名/アーティスト）→ ②ファイル名 の順で曲を特定します。タグが無い曲は、ファイル名がすべてです。

例判定
DNCE - Cake By The Ocean.mp3◎ アーティスト - 曲名
前前前世 - Natino Remix.mp3◎ 曲名 - リミキサー名 Remix
jzIWE3T_R21yFA1Uf....mp3✕ ランダム文字列＝何の曲か分からず失敗
… KLICKAUD.mp3 などサイト名入り△ 透かしは消しておくと検索が正確に
ファイル名を変えても音や中身には影響しません。この版はファイル名（またはタグ）だけを頼りにMVを探すので、名前がすべてです。

症状対処
「dj_maker_core.py が見つかりません」ファイルが同じフォルダに揃っていない。全部同じ場所に置く
「適切なアクセス権限がない」上の C/E（chmod +x）
「Need sudo access」/ Homebrew失敗上の 🔑の章（管理者にする。データは消えない）
「HTTP Error 403」/ 映像が見つからない上の 📡の章（Chromeでログインが一番効く）
MVが全然違う動画になる上の 🏷の章（名前を付け直す or URL版）
初回がすごく長い／止まって見えるAIライブラリのDL中（最大30分）。進捗が動いていれば正常。閉じないで待つ
Macのパスワードを聞かれた初回のHomebrew導入時だけ。macOSの仕様なので入力してOK
口の動き解析が出ないmediapipe未導入でも本体は従来通り動く（その機能だけ自動スキップ）
Intel Macで遅い対応済み。ただしAI処理（リップシンク曲）はApple Siliconより数倍かかります

💡 できること・出力されるもの

入れられる曲：MP3 / m4a / wav などの音声、mp4 などの動画。原曲も Remix / Edit も対応（1曲ベースのもの）。複数曲まとめてドロップで一気に書き出し。Mashup（複数曲ミックス）は非対応（どの曲のMVを使うか決められないため）。

自動でやること：公式MVをYouTubeで自動検索（再生数の多い本物を選択）／曲に映像を自動同期／歌っていない間奏に口パク映像が来ないよう調整。

出力されるMP4（DJ向け）：ビートグリッド書き込み済み（Serato / VirtualDJ に読み込むとBPMとグリッドが合った状態）／元のCueポイントを引き継ぎ（打ち直し不要）／1280×720・30fps／デスクトップに出力。

オプション：起動時の質問で Extended化（イントロ追加）／タイトルテロップの有無を選べます。

同じサビを何度も使い回すタイプのリミックスでは、口の同期が完璧にならない場合があります。その場合も全体として自然に仕上がるよう作っています。
配布時はこのフォルダごと渡してください（ファイルは全部同じ場所に置いたまま）。
